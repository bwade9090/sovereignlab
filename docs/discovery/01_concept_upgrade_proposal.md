# Concept upgrade proposal: K-VINTAGE on KOR-RTD

- Status: approved in full by Hyungbae on 2026-07-14; implemented as charter v2 and ADR 0003. This document is retained unmodified below as the frozen decision-rationale record — where it differs from the charter, the charter governs.
- Date: 2026-07-14
- Problem addressed: charter v1 is technically satisfying but reads as a technology-practice project — the idea is not novel and creates little added value outside the repository itself
- Provenance: multi-agent study (5 research agents — prior art, Mistral strategy, analyst pain points, internal critique, vintage-data feasibility; 6 ideation lenses; 7 consolidated candidates; per-candidate judge panels for feasibility / novelty / hiring impact, weighted 30/30/40). Facts labeled "verified" were checked against live endpoints or primary sources during the study. Items labeled "week-1 spike" must be re-verified before amending the charter.

## 1. Why charter v1 reads as a practice project

1. **The thesis noun is a "lab" and the predicate is a technique list.** Charter §1 states a real problem (point-in-time evidence for analysts) but resolves it into "an evaluation-first, bilingual Mistral deployment lab", and the AGENTS.md mission says the project "must prove fine-tuning, advanced RAG, deterministic tool use, evaluation, and production integration" — the job posting's checklist restated as a mission. The `closed_book / temporal_rag / rag_tools / lora_router` ladder is the canonical 2024–25 tutorial shape.
2. **The fictional customer is confessed in writing.** Charter §1: "The project is deliberately framed as a customer pilot." A pilot report with no counterparty friction, scope changes, or compromises is recognizable to anyone who has sat in a real pilot.
3. **The reverse-engineered origin is publicly auditable.** The gap matrix in `docs/discovery/00_role_gap_analysis.md` (fine-tuning 1/5 → `lora_router`; RAG 1/5 → temporal retrieval; …) maps one-to-one onto charter components. Honest, but it turns "technology practice" from a risk into documented intent.
4. **The genuinely rare assets are buried as metric rows.** As-of vintage correctness, ADR 0002's insight that "a current API response may contain revised historical observations", KR/EN parallel evaluation, and calibrated abstention are a central-bank economist's moat — yet they appear as single measurement lines, while the headline ("auditable cited briefing") is 2026 table stakes.

## 2. Recommended concept

Merge the two top-scoring candidates — **K-VINTAGE** (benchmark, headline) on **KOR-RTD** (point-in-time data layer, substrate) — and absorb the pilot-report retention condition from the "Customer Zero" candidate and the document-level sovereignty framing from "The Price of Sovereignty".

**New one-line pitch:**

> Build K-VINTAGE, a bilingual (KR/EN) macroeconomic QA benchmark in which the gold answer depends on the statistical data vintage that actually existed at the question's `as_of` date — to our knowledge the first such benchmark for official statistics — on top of KOR-RTD, the point-in-time data layer Korea does not have, with the SovereignLab pipeline as its reference implementation and public baseline suite.

**Flagship capability:** a deterministic as-of resolver — `as_of → max(EDITION ≤ as_of)` over the OECD SDMX EDITION dimension — that computes gold answers for "what did the data say *then*". Verified during the study: the live `DSD_STES_REVISIONS@DF_STES_REVISIONS` dataflow carries Korean editions from Feb-1999 to Jul-2026; the KOR CPI 2005-01 observation has 258 archived editions; Korea's 2025-Q1 real GDP was revised from 572,057.7 to 574,984.3 billion KRW between the June and July 2026 editions. A model answering from the latest value is caught mechanically (a "revision trap"), with no LLM judge involved.

**Headline public artifacts:**

- **K-VINTAGE benchmark** — 40 human-reviewed core questions (4 routes × KR/EN, unchanged contract) plus 200–300 machine-generated deterministic data-route probes; released as an HF dataset with a datasheet, a pip-installable grader, a **gold-answer regeneration script** (anyone can recompute from SDMX), and a PR-based static leaderboard seeded with the project's own four variants plus two API models. Always described as "40 human-reviewed + N machine-generated", never as "a 500-question benchmark".
- **KOR-RTD layer** — a tidy, documented consolidation of OECD edition histories for Korea, plus a scheduled GitHub Actions harvester (weekly, basket capped at ≤12 series) that forward-captures ECOS/KOSIS — both verified latest-only with no as-of parameter — into append-only, checksummed files under the frozen `SourceManifest` contract. Public commit history is independently verifiable proof of capture dates and of no backfill; vintages not captured from this week onward are lost to everyone forever, which is a moat no later competitor can clone.
- **Baseline suite** — the existing four-variant ablation, re-aimed: `closed_book` gets caught citing revised values; `rag_tools` with the as-of resolver targets ~0% leakage. **Temporal-leakage rate is promoted from a metric row to the headline number.**
- **Engagement report retained** (a condition from both hiring judges): the fictional-client narrative is replaced by a report grounded in the real artifacts, treating the archive as the data-readiness layer, plus a one-page mapping of the project's evaluation vocabulary to Mistral Studio/Forge terms (one table in one positioning document; no codebase renaming).

**Killer demo — "Right Then, Wrong Now" (90 seconds):** the same Korean question asked with `as_of=2026-06-15` returns an answer citing the June edition with checksum chips; asked with `as_of=today` it returns a different — equally correct — answer with the revision diff highlighted; a comparison card notes that the Bank of Korea's own public ECOS API has no as-of parameter and cannot answer the first question; the exchange is repeated in English. An interviewer can reproduce it with `curl`.

**Why it is novel (qualified, with prior art cited up front):** no existing LLM benchmark makes the gold answer depend on the data vintage available at a past date — MACROCAST (arXiv 2606.28670) uses vintage-aligned data but is pure time-series forecasting; contamination benchmarks (LLMLagBench, AntiLeak-Bench) measure model-knowledge leakage, not the vintage of the evidence record. The claim must always be stated as "**to our knowledge the first for official statistics**", citing the nearest cousins first: the German statutory as-of QA benchmark (arXiv 2605.23497, same abstract mechanism in the legal domain), the Dallas Fed real-time OECD dataset, and the OECD MEI revisions database (public vintage records including Korea). Never claim "first Korean macro benchmark" — KMMLU contains economics categories.

**Who gets real value:** (1) LLM-evaluation researchers — deterministic temporal-contamination probes needing no annotation; (2) Korea's sovereign-AI programs — evaluation harnesses are scarce; (3) Mistral Seoul — entered Korea via the NAVER Cloud partnership (2026-07-08) with no central bank or statistical agency on its public customer list; the baseline suite is a template for quantifying the frontier-API-vs-local-open-weights gap; (4) economists and Korea desks — the only public as-of query path for Korean macro data; the Croushore–Stark/Orphanides real-time research program is currently impossible for Korea with public tools (possible only for ~13 headline MEI indicators via the OECD revisions database — cite this precisely).

## 3. What stays unchanged

- **All frozen Pydantic v2 contracts**: `SourceManifest` applies verbatim to SDMX edition snapshots and BOK PDFs; `BenchmarkRecord`'s four exclusive routes, as_of cutoff validation, and evidence-group + KR/EN-parallel split-leakage checks become the benchmark's record format. ADR 0002's rejection of "observation date as proof of historical availability" is this concept's founding rationale, already written down.
- The 4-route taxonomy and evidence router (vintage questions are `data` / `documents_and_data` routes).
- The four-variant ablation ladder (renamed "baseline suite"; structure identical).
- The LoRA/QLoRA plan (Ministral 3 3B, rented GPU, frozen promotion rule, negative-results preservation) — survives as baseline #4.
- FastAPI service + trace UI, CI, spend ledger, offline-testable recorded-fixture design, staged CV-bullet disclosure rules.
- BOK document corpus scope: 58 Korean Economic Outlook reports (2012.07–2026.05, stable nttId URLs) + 13 English full translations (verified).
- All hard constraints: 4 weeks / ~80 h / solo, USD 100 ceiling, public data only, no implied endorsement, Windows / Python 3.12.

## 4. Proposed charter amendments

1. **§1 thesis rewrite**: delete "evaluation-first bilingual Mistral deployment lab" and "deliberately framed as a customer pilot". New thesis: the pitch above plus a falsifiable question — "does the system answer from the data vintage that existed at `as_of`, or does it leak the future — and can a compact open-weight router be non-inferior to a frontier API at that discrimination layer?"
2. **§2 persona replacement**: drop the four-institution OR. (a) Name the benchmark's four real consumer groups; (b) name the owner as **customer zero in a personal capacity**, with a dated dogfood log as a committed deliverable.
3. **§4 evidence-scope revision**: replace the plain OECD SDMX wrapper with the **vintage-resolving as-of tool** — primary: live `DSD_STES_REVISIONS@DF_STES_REVISIONS` (verified key-free and free of charge); cross-check: frozen `DF_MEI_ARCHIVE` (**week-1 spike — one verification returned 404, another queried it successfully**); forecast vintages: recent EO editions only (**live-confirmed range is DF_EO_114–118 plus current; do not claim "EO60–119" until the spike settles it**). Keep ECOS as the latest-only contrast tool. Record ALFRED as excluded (Korean vintage coverage shallow — GDP only from 2021, CPI feed discontinued 2024 — and the June-2024 FRED API terms prohibit archiving, redistribution, and ML/LLM use).
4. **§5 evaluation-contract extension**: two-tier gold set (40 human-reviewed core + 200–300 machine-generated probes, always reported separately); deterministic gold-answer computation recorded alongside human review; a **number-normalization spec (raw 10^12 XDC scale vs "billion KRW" presentation, 조/억 conventions, seasonal-adjustment variants, rounding tolerances) as a prerequisite artifact before question authoring**; temporal-leakage rate promoted to a headline per-baseline number. Korean CPI is essentially unrevised (50 identical editions verified) — use it as an abstain/trap item, never as a revision gold question.
5. **Schema-contract extension (ADR 0003)**: add a vintage/EDITION field to tool expectations. ADR 0002's revisit trigger #2 ("the SDMX tool contract needs source-vintage semantics not expressible by the current manifest") anticipates exactly this, so the extension is principled and likely non-breaking; record rationale and migration notes.
6. **§6 sovereign-mode promotion**: change "may use a small local router" to a **committed hybrid-mode demo** (local quantized router + API generation) with memory/latency recorded. One fully local end-to-end run is a documented stretch goal (without it, no "on-prem"-family CV wording — existing disclosure rule).
7. **§7 deliverables revision**: add the HF dataset + datasheet + grader + regeneration script + static leaderboard; the weekly harvester (basket ≤12 series; **per-series KOGL redistribution rulings are a week-1 gate — no raw values committed before the ruling; restricted series fall back to checksum-diff + fetch script**); live events: **primary = the 2026-08-01 OECD edition rollover (guaranteed — the July edition already contains new Korean GDP revisions); stretch = Korea's Q2-2026 advance GDP release (~2026-07-23/24, day 9–10 of the window — feasibility judges flagged the end-to-end path as tight)**; the pilot report is redefined (not deleted) as an engagement report grounded in real artifacts, plus the one-page Studio/Forge vocabulary mapping.
8. **Definition-of-done additions**: (a) ≥1 artifact whose creation time is independently verifiable from public CI logs; (b) the archive regenerates and validates from committed manifests; (c) every "first" claim follows the rule: "to our knowledge" + domain qualifier (official statistics) + prior-art citations (2605.23497, Dallas Fed, OECD MEI revisions DB).
9. **Korea-timing narrative paragraph** (§1 or the application docs): NAVER Cloud–Mistral partnership (2026-07-08, FDEs deployed in Korea), absence of central banks/economic-policy institutions from Mistral's public customer list, Korea's AI Basic Act (in force 2026-01) — **describe the Act precisely as regulating "high-impact" (고영향) AI with voluntary verification/certification**; calling it "high-risk" with mandatory testing is factually wrong and was the fatal flaw of a rejected candidate concept.
10. **Pre-committed cut ladder** (§7 or §9): on overrun, cut in this order — ① probe-set size (300→200→100) → ② forecast-vintage question family (defer to v1.1) → ③ leaderboard seed rows → ④ trace-UI polish. This makes mid-project de-scoping a plan rather than a failure.

## 5. Four-week plan impact

- **Week 1 (gate week)**: verification spikes — `DF_MEI_ARCHIVE` accessibility, actual EO edition range, per-series KOGL rulings, license status of OECD archive content published before 2024-07-01 (CC BY 4.0 is the default only for later publications), and a 1-hour written employer-risk review (personal-capacity public activity) — **all recorded in writing**. Vintage resolver (~6–8 h; reuses prototyped REST calls), harvester cron + manifest wiring (~3–5 h), number-normalization spec, charter amendment + ADR 0003, and the already-planned QLoRA compatibility spike.
- **Week 2**: author the 40 human-reviewed questions (**budget 8–12 h honestly — machine generation does not remove this bottleneck**), generate the probe set + sample human audit, retrieval baselines, minimal end-to-end path. Stretch: Q2 GDP release note (report the automation level honestly — promise "a cited note within 48 h", not "same-day").
- **Week 3**: LoRA — **hyperparameter matrix reduced to 2–3 configurations (a committed cut)**, judged by the frozen promotion rule; four-baseline evaluation with the leakage headline number; **2026-08-01 edition-rollover live event**.
- **Week 4**: HF release + grader packaging + leaderboard seeding, demo script (reproducible from one seeded command), engagement report + Studio/Forge mapping, runbook, CV-bullet placeholder resolution.
- **Committed de-scopes**: fictional-pilot narrative writing (recovers 4–6 h), full deployment-mode matrix, deep EO backfill (recent editions only), twice-daily harvester cadence, a separate sibling repo (use a `data/` layer in this repo), trace-UI polish (bottom of the cut ladder).
- **Budget**: data $0 (all endpoints verified key-free and free), API ~$25, GPU ~$50 — ceiling unchanged. Net new work is **+25–35 h, absorbed by the de-scopes and the cut ladder; there is no slack.** If a weekly gate is missed (week-1 spike results; end-to-end by end of week 2), invoke the ladder immediately.

## 6. Alternatives considered (not chosen)

- **Customer Zero + "Two Futures, One Quarter" (7.0)** — replace the fictional pilot with auditable real usage and a real case study (the EO107 double-scenario edition). Most honest fix for the "cosplay pilot" problem, but it is evidence of use rather than a reason to exist (novelty 6), and its live feed collapses if the employer gate fails. Its dogfood log and pilot-report retention were absorbed into the recommendation; the EO107 reconstruction remains a good post-window case study.
- **Outlook Diff Desk / BOK Watch (6.8)** — event-driven, revision-aware bilingual diff briefings on BOK/OECD outlooks. Strongest demand evidence of all candidates (FOMC-diff practice), but worst feasibility (net +35–65 h), the sharpest employer risk (a serving BOK employee auto-publishing a public tracker of BOK communication), and its marquee event (the next full outlook) falls outside the window. Its core assets (revision-join, edition verification) are already inside the recommended vintage layer; keep as the post-window flagship application.
- **The Price of Sovereignty (6.35)** — measured cloud-vs-local trade-off study with a CTO-grade decision brief. Novelty 4.5 (the genre exists); its lossless floor (6–10 h of document reframing) was absorbed into the engagement report.
- **asofguard (5.9)** — pip-installable temporal-leakage linter for RAG. Nice packaging, weak moat; the regeneration script + grader cover most of the value.
- **Governance-in-a-Box (5.25, fatal flaw)** — rejected: its load-bearing claim misstates Korea's AI Basic Act (the Act regulates "high-impact" AI and imposes no mandatory pre-market testing; verification/certification is voluntary).

## 7. Top risks and mitigations

1. **Schedule overrun (the dominant risk)** — both top concepts scored "ships only with discipline, no slack". → Pre-committed cut ladder, LoRA matrix fixed at 2–3 configs, 8/1 rollover as the primary event, weekly gates. Partial delivery still lands honestly under the staged-bullet rules.
2. **"First" overclaim** — the German statutory as-of benchmark (2605.23497) is the same abstract mechanism in law; Dallas Fed and the OECD MEI revisions DB are public vintage records including Korea. → Qualify every claim ("official statistics, to our knowledge") and cite the prior art in the datasheet **before** a hostile reviewer finds it. Drop any "first Korean macro benchmark" phrasing (KMMLU has economics categories).
3. **Data-endpoint instability** — conflicting `DF_MEI_ARCHIVE` results during the study; EO live range confirmed only 114–118; a `NonProductionDataflow` annotation was observed on the live revisions flow. → Week-1 spike settles the claimable backfill range; committed `SourceManifest` checksum snapshots keep the benchmark reproducible even if OECD reorganizes endpoints; disclose the annotation in the datasheet.
4. **Grading noise** — raw SDMX values are on a 10^12 XDC scale vs "billion KRW" presentation; 조/억 conventions; series variants (B1GQ_D/Q/V). → Freeze the normalization spec before question authoring (§4-4); explicit tolerances; unrevised Korean CPI used only as a trap.
5. **Employer and licensing sensitivity** — a serving BOK employee publishing publicly, "BOK's API can't do this" framing, KOGL/OECD licenses. → Standing personal-capacity disclaimer; frame as **"complementing public statistical infrastructure"**, never "pointing out defects"; commit the written week-1 risk review as a decision record (itself evidence of regulatory sense); no raw values before per-series KOGL rulings; auto-generated notes labeled strictly as "automated reconstruction of official releases" (not market commentary or advice).

## 8. Draft CV bullets (staged-disclosure compliant)

- **Designing K-VINTAGE**, a bilingual (Korean/English) macroeconomic QA benchmark in which gold answers are computed from the statistical data vintage available at each question's as-of date — to our knowledge the first such benchmark for official statistics — with deterministic grading from OECD SDMX edition histories, evidence-disjoint splits, four evidence-route labels including calibrated abstention, and a public gold-answer regeneration script.
- **Building KOR-RTD**, a provenance-contracted point-in-time data layer for Korean macroeconomics: consolidated OECD edition histories with per-snapshot SHA-256 checksums and per-source licensing decisions, plus a scheduled public harvester that forward-captures Korea's official statistics APIs (ECOS/KOSIS), which expose latest values only — commit history serving as independently verifiable proof of capture dates.
- **Building the reference briefing service and baseline suite** on this layer (temporal RAG + deterministic vintage tool + QLoRA-tuned Ministral 3 router evaluated under a pre-frozen promotion rule), reporting temporal-leakage rate as a deterministically verified headline metric; measured results to be inserted only after evaluation runs complete, per the repository's staged-disclosure rules.

## 10. One-line conclusion

Keep the pipeline; change the project's noun. Not "a Mistral deployment lab", but **"the point-in-time data layer Korea doesn't have, the first vintage-conditioned benchmark that is only possible on top of it, and the first operational system that consumes it."** The frozen schemas (ADR 0002) already anticipated this direction — this is a promotion, not a pivot.
