# Architecture decision records

Use ADRs for decisions that materially affect reproducibility, evaluation validity, cost, security, public data rights, or system architecture.

File naming: `NNNN-short-kebab-case-title.md`.

Each ADR should contain:

- Status and date.
- Context and constraints.
- Decision.
- Alternatives considered.
- Consequences and an explicit revisit trigger.

Accepted records are not silently rewritten. Supersede them with a new ADR and cross-link both records.
